# Assignment Analysis

## 1. Selected route

- AI paradigm: `C: Multi-agent`
- Base LLM: `Qwen3-Max`
- Integration framework: `Spring AI Alibaba`

## 2. Mandatory deliverables from the PDF

The assignment requires three concrete deliverables for the chosen route:

1. Source code
2. Complete conversation log of the interaction with the LLM, with timestamps, across four iterations
3. Report in English

## 3. Constraints that shape the implementation

The multi-agent option explicitly requires:

- prior knowledge to be provided to the AI,
- multiple role prompts,
- dialogue rules,
- workflow definition,
- Mermaid or PlantUML for views,
- no external domain knowledge beyond the provided prior knowledge,
- no few-shot examples,
- no hidden rule base,
- and collaborative decomposition plus verification derived only from the system instructions.

## 4. How this repository maps to the assignment

### Prior knowledge

The shared content is stored in:

- `src/main/resources/knowledge/add-3.0.md`
- `src/main/resources/knowledge/hotel-pricing-system.md`
- `src/main/resources/knowledge/iteration-plan.md`

### Multiple role prompts

The role prompts are generated by `PromptFactory` for:

- Requirements Analyst
- Solution Architect
- Quality Reviewer
- Diagram Curator
- Iteration Moderator

### Dialogue rules

Each role prompt includes explicit rules for:

- using only the provided prior knowledge,
- citing driver IDs,
- avoiding external assumptions,
- preserving iteration scope,
- and producing outputs in English.

### Workflow

Each iteration executes this fixed collaborative sequence:

1. Analyst brief
2. Architect draft
3. Reviewer verification
4. Architect revision
5. Diagram generation
6. Moderator consolidation

### Conversation log

Every agent exchange is recorded with:

- timestamp,
- system prompt,
- user prompt,
- model response,
- latency,
- token metadata if available.

### Report

The generated `report.md` follows the appendix structure and leaves the individual reflection section editable for the team.

## 5. Recommended submission flow

1. Set the DashScope API key and enable DashScope.
2. Run the application.
3. Trigger `POST /api/add/runs`.
4. Review the produced `artifacts/<run-id>/report.md`.
5. Polish the English wording if needed.
6. Complete the individual reflection section with real team contributions.
